# 流水线两侧面板折叠按钮优化设计

## 背景

流水线编辑器左右两侧面板当前收起后仍保留约 48px 的宽轨道，圆形箭头按钮视觉重量较大，压缩画布可用空间且缺少明确的面板边界。需要在不改变面板功能、拖拽调宽和运行区行为的前提下，优化展开/收起按钮。

## 目标

- 收起后的侧栏占用尽可能少的画布空间。
- 展开与收起方向一眼可辨，左右两侧行为保持对称。
- 按钮具备清晰的 hover、focus 和 active 反馈，并与现有深色编辑器视觉语言一致。
- 保留现有面板宽度拖拽、双击恢复默认宽度和响应式布局逻辑。
- 不改变节点属性、运行日志、参数配置和画布数据行为。

## 方案

采用“极窄边缘把手”方案：

- 展开状态：按钮贴近侧栏内侧边缘，使用低高度、窄宽度的胶囊把手，不再使用独立大圆按钮。
- 收起状态：侧栏宽度从 48px 缩减至约 26px，仅保留一条带有箭头图标的边缘把手。
- 左侧收起后显示向右箭头，右侧收起后显示向左箭头；展开时图标反向表示收起动作。
- hover 时使用主题强调色、轻微阴影和横向位移动画；focus-visible 时显示可访问的轮廓。
- 把手使用伪元素或窄面板背景形成贴边层次，避免看起来像悬空按钮。

## 交互与状态

- 点击把手切换对应面板的 `leftCollapsed` 或 `rightCollapsed` 状态。
- 收起状态不渲染面板内容和拖拽调宽手柄，展开后恢复原有内容。
- 右侧面板收起不影响底部运行抽屉的日志和状态展示。
- 保持原有 title 文案，补充 `aria-label`，键盘可通过 Tab 聚焦并按 Enter/Space 操作。

## 实现边界

- 仅修改 `PipelineEditorView.vue` 中两侧面板的模板结构与 CSS。
- 不调整运行时数据、节点属性数据、持久化接口或画布节点布局。
- 复用现有 CSS 变量和过渡时长，避免新增依赖。

## 验证

- 定向执行 ESLint。
- 执行生产构建，确认 Vue 模板和 CSS 编译通过。
- 检查收起状态的面板宽度、按钮可点击区域、焦点样式及运行抽屉左右边界。
