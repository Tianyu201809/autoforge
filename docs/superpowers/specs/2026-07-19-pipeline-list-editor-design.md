# 流水线列表页与编辑页拆分设计

## 目标

将当前“列表、编辑器、节点库、运行状态”全部堆叠在一个视图中的流水线弹窗，拆分为清晰的列表页和编辑页，同时改善页面过渡、节点操作反馈和编辑器整体布局。保持现有流水线数据模型、IPC 接口和执行协议不变。

## 范围

- 流水线入口仍使用现有大弹窗，不引入全局路由或独立窗口。
- 弹窗打开后默认显示流水线列表页。
- 列表页支持关键词搜索，以及“全部 / 收藏 / 归档”筛选。
- 列表项展示名称、描述、节点数量、最近运行时间和当前运行状态。
- 列表页提供新建、编辑、收藏/取消收藏、归档/取消归档和删除入口。
- 新建和编辑均进入独立的编辑页；编辑器继续支持节点编排、参数映射、保存、运行和停止。
- 使用 GSAP 编排页面、列表卡片、节点增删和检查器折叠动画，并遵守 `prefers-reduced-motion`。

不在本次范围内：全局路由改造、流水线执行协议调整、条件分支/并行节点/循环节点、历史记录新页面、后端数据表结构变更。

## 方案与组件边界

采用“弹窗壳 + 两个内部页面组件”的方案。

### PipelinePanel（弹窗壳）

- 持有 `view: 'list' | 'editor'`、当前流水线 id、编辑模式和全局加载状态。
- 负责加载流水线、环境和运行会话，订阅流水线会话事件。
- 负责打开/关闭弹窗、页面切换、返回确认和子组件事件转发。
- 提供统一的页面标题区和 GSAP 页面切换容器。

### PipelineListView（列表页）

- 接收流水线清单、运行会话和加载状态。
- 本地计算关键词匹配与收藏/归档筛选结果，不修改后端查询协议。
- 发出 `create`、`edit`、`toggle-star`、`toggle-archive`、`delete` 事件。
- 使用卡片列表展示清单；空状态分别覆盖“没有流水线”和“筛选无结果”。
- 列表卡片使用 GSAP stagger 入场，筛选变化使用可回收的离场/入场过渡。

### PipelineEditorView（编辑页）

- 接收脚本、环境、当前草稿、运行会话和保存状态。
- 复用现有无限画布、脚本库、节点检查器、输入映射与运行控制逻辑。
- 发出 `back`、`save`、`run`、`stop`、`delete` 等事件。
- 编辑页顶部提供返回列表、面包屑、保存、运行/停止和删除操作。
- 节点增删、选中态、左右侧栏折叠使用 GSAP 过渡；画布连接线继续沿用现有 SVG 动画。

### 共享状态

优先在 `PipelinePanel` 中保留现有数据加载和 IPC 调用，子组件通过明确的 props/emits 通信。若拆分后单个状态块仍过大，再抽取 `usePipelineEditor`，但不新增全局状态库。

## 页面与交互流程

1. 用户从侧边栏打开流水线，弹窗进入列表页并加载清单。
2. 点击“新建流水线”时创建空白本地草稿，切换到编辑页；只有保存后才写入后端。
3. 点击列表卡片或“编辑”进入编辑页，加载该流水线副本作为草稿。
4. 编辑过程中修改名称、描述、节点顺序、节点参数和映射；保存成功后更新当前草稿和列表数据，并保留在编辑页。
5. 点击“返回列表”时，如果草稿与最近保存版本不同，显示未保存确认；确认后放弃草稿并返回列表。
6. 删除成功后返回列表并刷新清单；归档/收藏操作在列表页即时更新，并回写后端。
7. 运行中的流水线在列表卡片和编辑页同时显示状态；编辑页保留停止能力。
8. 关闭弹窗沿用现有入口行为；若存在未保存草稿，复用确认机制阻止误关闭。

## 列表查询与展示

- 关键词同时匹配 `name` 和 `description`，忽略大小写并做前后空白裁剪。
- 筛选器：
  - `全部`：显示未归档和已归档流水线。
  - `收藏`：只显示 `starred === true` 的流水线。
  - `归档`：只显示 `archived === true` 的流水线。
- 默认排序保持后端名称排序；前端不引入不稳定的时间排序。
- 最近运行时间使用 `recentRunAt`，无记录时显示“尚未运行”。
- 运行状态从当前 `PipelineSession` 映射：运行中、成功、失败、已停止；没有活动会话时显示空闲。

## 动画与可访问性

- 页面切换：编辑页从右侧淡入，列表页从左侧淡入；切换期间保持容器尺寸稳定，避免弹窗跳动。
- 列表卡片：首次加载和筛选变更使用 `opacity` + `transform` 的短时 stagger 动画。
- 编辑器：节点添加后淡入并轻微上移，删除时先缩放淡出；检查器和侧栏折叠使用宽度/透明度过渡。
- GSAP 统一管理需要时间线的动画，组件卸载时调用 `context.revert()` 或显式清理。
- `prefers-reduced-motion: reduce` 下禁用 stagger、位移和循环装饰动画，仅保留即时状态变化。
- 所有图标按钮保留 `title` 和可访问名称；页面切换后将焦点放到标题或主操作按钮。

## 错误处理

- 列表加载失败：保留弹窗，展示错误提示和重试按钮，不进入空列表误导用户。
- 保存失败：保留编辑草稿，展示错误信息，允许再次保存。
- 新建草稿为空时点击运行/保存：沿用现有校验，提示名称和至少一个节点要求。
- 删除、归档、关闭和返回涉及未保存内容时先确认；取消后保持当前页面和草稿。
- 选中的脚本被删除或不可用时，编辑页继续显示节点错误状态，保存/运行沿用现有校验提示。

## 验证范围

- 列表页默认打开、关键词搜索、全部/收藏/归档筛选和空状态。
- 新建进入编辑页，保存后回到列表可见；编辑、删除、归档、收藏状态正确刷新。
- 未保存返回和关闭确认不会丢失草稿。
- 编辑器现有节点增删、排序、映射、保存、运行、停止行为回归。
- 列表卡片与编辑页运行状态跟随 `onSession` 更新。
- GSAP 动画在正常模式和减少动态效果模式下均不阻塞操作。
- `npm run lint` 与 `npm run build` 通过。
