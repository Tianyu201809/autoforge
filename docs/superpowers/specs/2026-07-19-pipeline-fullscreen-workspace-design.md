# 流水线全屏工作区重构设计

## 目标

将流水线从弹窗升级为主窗口内的独立全屏工作区，隐藏脚本主界面的全局侧栏和弹窗遮罩，提供更适合编排任务的“列表页 + 三栏编辑器”体验。视觉方向参考用户提供的工作流产品截图中的信息层级和工作区分区，但采用 Autoforge 自有的浅色主题、脚本模型和操作语言。

## 范围

- 流水线入口改为主窗口内容状态切换，不再使用固定尺寸弹窗。
- 全屏工作区包含流水线列表页和流水线编辑页。
- 列表页保留关键词搜索、全部/收藏/归档筛选、最近运行时间和运行状态。
- 编辑页采用左侧节点库、中间无限画布、右侧节点检查器三栏布局。
- 顶部工作区栏提供返回主应用、列表/当前流水线面包屑、保存、运行/停止和收藏操作。
- 流水线输入参数与运行状态放入底部可折叠抽屉，避免挤压画布和检查器。
- 保留现有 Pipeline IPC、数据库结构、节点数据模型、参数映射和运行协议。
- 保留未保存修改确认、减少动态效果支持和现有 GSAP 动画能力。

不在本次范围内：新增 Vue Router、独立 Electron BrowserWindow、修改流水线后端协议、条件分支/并行节点/循环节点、历史记录独立页面。

## 页面承载与导航

### App Shell

`App.vue` 继续作为主窗口根组件，但由 `showPipelines` 决定渲染普通脚本工作台还是 `PipelineWorkspace`。进入流水线时：

- 隐藏主应用的脚本侧栏、主内容区和弹窗遮罩。
- 流水线工作区占据主窗口客户区，不改变 Electron 窗口本身。
- 工作区的“返回列表/返回主应用”操作恢复普通脚本工作台。
- 关闭主窗口的现有生命周期保持不变；工作区关闭前复用未保存确认。

### PipelineWorkspace

工作区负责流水线数据、运行会话、页面状态和子组件通信。页面状态仍采用 `list | editor`，不引入全局状态库。

- 列表状态：展示 `PipelineListView`。
- 编辑状态：展示 `PipelineEditorView`，通过 pipeline id 或空草稿进入。
- 保存、删除、收藏、归档后更新本地清单并刷新主应用脚本状态。
- 运行会话事件同时更新编辑页运行抽屉和列表页卡片状态。

## 视觉结构

### 顶部工作区栏

- 左侧：返回主应用按钮、`流水线 / 当前名称` 面包屑。
- 中间：保存状态（已保存/有未保存修改）。
- 右侧：收藏、保存、运行/停止、关闭或更多操作。
- 高度固定，保持轻量玻璃质感；不随画布缩放或拖拽改变。

### 列表页

- 页面标题区说明流水线用途，并提供新建按钮。
- 工具栏提供名称/描述搜索、全部/收藏/归档筛选和数量统计。
- 卡片展示名称、描述、节点数量、最近运行时间和运行状态。
- 卡片操作包括编辑、收藏/取消收藏、归档/取消归档和删除。
- 空状态分别覆盖没有流水线、筛选无结果和加载失败。

### 编辑页三栏

- 左栏：脚本节点库，提供搜索、分类标题、点击添加和拖拽提示；宽度固定且不受画布缩放影响。
- 中栏：无限画布，包含网格、适配、重置、缩放工具；画布区域负责平移和连接线动画。
- 右栏：节点检查器，展示节点名称、说明、固定参数、输入映射和删除操作；宽度固定且不受画布缩放影响。
- 底部抽屉：显示运行状态、节点完成情况、流水线输入参数和环境选择；默认收起，展开后覆盖画布底部而不改变三栏宽度。

### 画布缩放与平移

- 缩放仅作用于画布世界层和连接线，不作用于顶部工具栏、左侧节点库、右侧检查器、底部抽屉或编辑器外层盒子。
- 画布世界使用独立的 transform/缩放状态；外层编辑器和三栏布局使用 `min-width: 0`、固定侧栏宽度和内部溢出裁剪，避免缩放内容撑开主窗口。
- 右键拖拽支持八方向平移，左键拖拽保持兼容；右键菜单被画布拦截。
- 重置和适配操作只重置画布世界的缩放/平移，不改变 UI 布局。

## 组件与数据边界

### PipelineWorkspace

- 持有 `pipelines`、`sessions`、`view`、`editingPipeline`、加载和错误状态。
- 调用现有 `window.autoforge.pipelines.*` API。
- 处理列表操作、编辑页导航和运行事件订阅。

### PipelineListView

- 只接收清单、会话和加载状态。
- 本地完成搜索/筛选与卡片渲染。
- 通过 emits 发出创建、编辑、收藏、归档、删除和重试事件。

### PipelineEditorView

- 接收脚本、流水线草稿和当前会话。
- 管理节点编排、画布平移缩放、参数映射、检查器、保存、运行和停止。
- 通过 emits 发出返回、关闭、保存、删除和会话更新事件。
- 保持现有节点模型与 IPC payload 结构，不把 UI 状态写入后端。

## 交互与动效

- 列表进入工作区时使用卡片 stagger；列表/编辑切换使用短时淡入和水平位移。
- 节点添加、删除和检查器内容切换使用 GSAP；连接线继续使用轻量 SVG 动画。
- 画布缩放和平移使用 transform，不触发布局重排。
- `prefers-reduced-motion: reduce` 下关闭 stagger、位移和循环装饰动画。
- 所有侧栏、抽屉和工具按钮保留 title/可访问名称；页面切换后聚焦主标题或搜索框。

## 错误与恢复

- 工作区加载失败时保留全屏壳，展示重试状态，不回退成空列表。
- 保存失败保留草稿和画布位置，允许重新保存。
- 删除/归档/返回/关闭涉及未保存内容时先确认。
- 脚本引用失效时在节点卡片和检查器显示不可用状态，沿用现有保存/运行校验。
- 从编辑页返回列表后刷新清单和运行会话，但不影响普通脚本工作台当前选中项。

## 验证范围

- 主应用进入流水线后全屏替换，不出现弹窗遮罩或主脚本侧栏。
- 列表页搜索、筛选、新建、编辑、收藏、归档和删除行为。
- 编辑页三栏固定宽度；在不同画布缩放值下顶部栏、左节点库、右检查器和底部抽屉不被挤压。
- 画布缩放、右键八方向平移、连接线定位、适配和重置行为。
- 节点增删、排序、输入映射、保存、运行、停止和运行状态抽屉回归。
- 未保存返回/关闭确认、加载失败重试和减少动态效果模式。
- 运行 `npm run lint`（记录并区分仓库既有错误）与 `npm run build`。
