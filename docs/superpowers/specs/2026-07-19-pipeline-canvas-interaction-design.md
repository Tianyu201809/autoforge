# 流水线画布交互优化设计

## 目标

优化流水线编辑画布的密度、缩放一致性、字段连线和节点交互，使其更接近流程图编辑器，同时保持现有线性流水线执行模型与保存协议不变。

## 范围

- 缩小流水线节点卡片与卡片间连接器，提升单屏可见节点数量。
- 让节点卡片、字段和 SVG 连线共享同一个画布世界缩放。
- 增加“上游完整输出 → 紧邻下游输入字段”的主动端口连线。
- 修复输入字段、输入控件与画布平移之间的指针事件冲突。
- 保留脚本库拖入画布、现有节点排序、输入映射编辑、保存和运行能力。

不在范围内：新增脚本输出 schema、修改 `PipelineNode`/IPC/数据库协议、条件分支、并行节点、跨越节点连接或引入第三方流程图库。

## 数据流与持久化

连线完成后直接复用现有 `PipelineInputMapping` 数据结构：

```ts
{
  source: 'previous-result',
  sourcePath: '',
  targetParam: '<目标参数 key>'
}
```

空的 `sourcePath` 表示把上一个节点的完整返回值传递给目标参数。由于运行器按 `order` 顺序执行并使用 `previousResult`，画布只允许当前节点输出连接到紧邻的下一个节点输入字段。对同一目标字段再次连线时替换已有映射；删除映射仍通过映射行的删除按钮完成。无需修改共享类型、IPC 或运行器。

## 组件与交互设计

实现集中在 `src/renderer/src/components/PipelineEditorView.vue`：

1. **连接状态**
   - 增加连接草稿状态，记录源节点、源端口和当前指针位置。
   - 点击节点“完整输出”端口进入连接态。
   - 指针移动时绘制从源端口到指针位置的临时贝塞尔路径。
   - 紧邻下一个节点的输入端口显示可连接高亮；点击后写入映射并生成正式路径。
   - 点击空白、按 `Esc`、指向非法节点或取消指针操作时清理连接草稿。

2. **端口呈现**
   - 完整输出端口和输入字段端口默认低对比度，仅在字段/卡片悬停或连接态显示。
   - 连接态突出源端口、合法目标端口和临时路径；非法目标不响应完成操作。
   - 移除字段的原生 HTML5 `draggable`，避免拖拽幽灵图、输入框抢占和缩放坐标误差；脚本库拖入画布继续使用原生拖拽。

3. **画布平移与控件事件**
   - 画布指针平移仅在空白区域开始。
   - 节点卡片、端口、字段、按钮、输入框和选择框阻止平移冒泡。
   - 连接态优先于平移态，端口操作不会触发画布拖动。

## 视觉与缩放

- 节点卡片目标尺寸约为 `260px` 宽、`260px` 最小高度，并同步收紧内边距、字段行高和卡片间距。
- 节点卡片与 `workflow-connections` SVG 放入同一画布世界层，使用同一个平移/缩放变换；移除卡片上的 `zoom: 1 / canvasScale` 抵消样式。
- 缩放控制范围约为 `50%–150%`，按钮、`Ctrl + 滚轮`、适配和重置共用同一状态。
- 工具栏、左右侧栏、运行抽屉和编辑器外层不参与画布缩放。
- 正式连线使用平滑贝塞尔曲线、轻量光晕和现有运行状态颜色；临时连线使用强调色。`prefers-reduced-motion: reduce` 时禁用流动动画。
- 卡片间顺序箭头保留，作为执行顺序提示；字段连线用于表达具体数据映射。

## 错误处理与边界

- 没有选中源端口时，点击目标端口不创建映射。
- 目标节点不是源节点的紧邻下一个节点时，端口保持不可用并拒绝连接。
- 目标字段已存在映射时，连接新源会原地替换，避免一个参数产生多条隐式值。
- 节点删除或排序后重新测量所有路径；不存在的 DOM 端口不绘制路径，不抛出错误。
- 保存失败保留当前草稿和画布视图；已有的保存/运行错误提示继续复用。

## 验证

实现后按以下顺序验证：

1. 三个及以上节点加入后，默认视口能看到更多节点，卡片尺寸保持紧凑。
2. 调整缩放时卡片、字段、正式连线和临时连线同步缩放，工具栏和侧栏尺寸不变。
3. 从节点输出端口连到下一个节点输入端口，映射行和 SVG 路径同时出现；重复连接会替换；删除后路径消失。
4. `Esc`、点击空白和非法目标均能取消连接，不触发画布平移。
5. 各节点输入框可正常编辑，空白区域仍可平移，脚本库仍可拖入画布。
6. 保存后关闭并重新打开流水线，映射与节点顺序保持一致。
7. 运行 `npm run lint` 和 `npm run build`，记录并区分仓库已有问题与本次改动问题。
