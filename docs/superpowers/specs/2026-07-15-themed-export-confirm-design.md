# 主题化导出确认框 — 设计规格

**日期：** 2026-07-15  
**状态：** 已确认设计

## 目标

将脚本 ZIP 导出前的 Electron 原生确认提示替换为 Autoforge 现有主题化确认框。系统文件保存选择器继续使用 `dialog.showSaveDialog`。

## 交互

1. 用户在脚本卡片菜单选择“导出 ZIP”。
2. 渲染进程请求主进程生成导出预览。
3. 预览成功后，使用现有 `ConfirmDialogHost` 显示：
   - 标题：“导出脚本 ZIP”；
   - 文件数量与总体积；
   - 依赖包、运行产物、密钥和业务数据不会包含在 ZIP 中；
   - 动态加载资源需通过 `autoforge.json` 的 `export.include` 声明。
4. 按钮为“取消”和“选择保存位置”。
5. 用户确认后调用导出 IPC，主进程重新计算并校验清单，再打开系统保存选择器。
6. 用户取消主题确认框或系统保存选择器时均静默结束。
7. 预览或实际导出失败使用现有主题 Toast 显示错误；成功仍显示现有成功 Toast。

## 架构

- 新增 `SCRIPTS_EXPORT_PREVIEW` IPC，返回 `fileCount`、`totalBytes` 和可展示的 `message`。
- `SCRIPTS_EXPORT_ZIP` 删除 `dialog.showMessageBox`，保留清单重算、`showSaveDialog` 和 ZIP 写入。
- preload 与渲染进程类型增加预览方法和结果类型。
- `ScriptCard` 的 `handleExport` 依次执行 preview → `askConfirm` → export。
- 不新建弹框组件，不改变全局确认框视觉样式；自动复用当前主题变量、过渡、键盘与遮罩行为。

## 一致性与安全

- 预览结果仅用于展示，不作为最终写入依据，避免确认期间文件变化导致过期清单被使用。
- 实际导出仍执行全部白名单、路径、符号链接、文件类型和体积校验。
- 预览不写文件，也不打开系统对话框。
- 本次不改变 Hub 重复安装使用的原生主进程确认框；该流程由外部 HTTP 请求触发，不在本优化范围内。

## 验证

- 导出前不再调用 Electron `dialog.showMessageBox`。
- 深色、浅色及其他皮肤下确认框使用现有主题样式。
- 确认框显示正确文件数和大小。
- 取消确认不会打开保存选择器；确认后保存选择器正常打开。
- 预览后工作区发生变化时，实际导出会重新校验。
- 构建和相关文件 ESLint 通过。
