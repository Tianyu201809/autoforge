# Autoforge v1.19.0 版本说明

> 发布日期：2026-07-19 · 应用版本 `1.19.0` · 脚本规范 `autoforge 1.0`

Autoforge 是本机自动化脚本锻造与管理桌面应用。v1.19.0 新增 **AutoforgeHub 入口配置**：在设置中填写 AutoforgeHub 网站地址后，可从侧边栏一键在浏览器中打开 Hub。

完整功能基线见 [v1.9.0](./v1.9.0.md)；近期 Hub 安装与导出见 [v1.15.0](./v1.15.0.md)、[v1.18.0](./CHANGELOG.md)。本文档仅描述 v1.19.0 新增与变更。

实现计划：[2026-07-19-autoforgehub-entry.md](./superpowers/plans/2026-07-19-autoforgehub-entry.md)

## AutoforgeHub 入口配置

### 前提

v1.15.0 引入了本机 HTTP 桥（`127.0.0.1:19276`），使 Hub 网站可探测并请求安装脚本到本机 Autoforge。但用户在桌面端一直没有从应用到 Hub 的直达路径。v1.19.0 补齐了这一环节。

### 设置面板

在设置面板中新增独立的 **AutoforgeHub** 配置区：

- 输入 AutoforgeHub 网站地址（支持 `http://` 和 `https://`）
- 保存时自动去除首尾空白
- 地址持久化到本地 SQLite 配置（`AppConfig.hub.url`），无需新增数据库迁移

### 侧边栏入口

侧边栏原有的「脚本市场」（带 `Soon` 标记的占位按钮）替换为 **「进入AutoforgeHub」** 功能按钮：

| 场景 | 行为 |
|------|------|
| 已配置地址 | 在系统默认浏览器中打开 AutoforgeHub |
| 地址为空 | toast：「AutoforgeHub地址未设置」 |
| 地址无法打开 | toast：「AutoforgeHub地址无效」 |

点击后通过 `system.openExternal` 在系统浏览器中打开，不在应用内嵌入网页。

### 模块

| 模块 | 职责 |
|------|------|
| `AppConfig.hub` | 共享类型定义（`hub?: { url?: string }`） |
| `ConfigRepository` | 深度合并 `hub` 配置，保证与现有字段不冲突 |
| `SettingsPanel.vue` | 设置面板中 Hub 地址的输入、加载、保存 |
| `Sidebar.vue` | 侧边栏「进入AutoforgeHub」按钮与 toast 反馈 |

本次未新增 IPC 通道、数据库迁移或 Hub 专有服务，全部复用现有基础设施。

---

## 与 Hub 安装管线的配合

v1.19.0 的入口配置与 v1.15.0 / v1.18.0 构建的 Hub 安装管线形成闭环：

```
设置 AutoforgeHub 地址 → 侧边栏「进入AutoforgeHub」→ 浏览器打开 Hub
    ↓
Hub 网站浏览脚本 →「添加到本地」→ POST /install → 本机导入
```
