# Autoforge v1.10.0 版本说明

> 发布日期：2026-07-03 · 应用版本 `1.10.0` · 脚本规范 `autoforge 1.0`

Autoforge 是运行在 Windows 上的 Electron 桌面应用，作为**本机自动化脚本锻造与管理平台**。v1.10.0 在 v1.9.0 能力基线上，重点解决**开发模式与安装包/预览模式数据混用**的问题，并加强手动运行的误触防护。

完整功能清单见 [v1.9.0 版本说明](./v1.9.0.md)；本文档仅描述 v1.10.0 新增与变更。

---

## 相对 v1.9.0 的变化

| 领域 | 变更 |
|------|------|
| **数据隔离** | `npm run dev` 与 `npm run build` / 安装包使用独立的 userData 目录 |
| **自动迁移** | 旧版 `%APPDATA%/autoforge/` 扁平数据首次启动时复制到 `-development` / `-production` |
| **路径修复** | 迁移后自动重写 SQLite 与 JSON 配置中的绝对路径，修复错误累积的 `workspace_path` |
| **运行确认** | 详情面板与卡片快捷运行前弹出二次确认对话框 |
| **构建模式** | `electron-vite` 区分 `development` / `production` mode；新增 `npm run prod` |

---

## 运行环境 vs 脚本环境 Profile

请勿混淆两个「环境」概念：

| 概念 | 含义 | 示例 |
|------|------|------|
| **应用运行环境**（`AppEnv`） | 由启动方式决定：`dev` → development，`build`/安装包 → production | 决定 userData 目录 |
| **脚本环境 Profile** | 用户在设置中配置的 dev/test/prod 变量集 | 决定 `ctx.env` 取值 |

---

## 本机数据目录（v1.10.0）

Windows 下位于 `%APPDATA%/`（Roaming）：

| 目录 | 何时使用 |
|------|----------|
| `autoforge-development/` | `npm run dev` |
| `autoforge-production/` | `npm run build`、`npm run preview`、`npm run dist:win` 及 NSIS 安装包 |
| `autoforge/` | 旧版扁平布局（v1.10.0 首次启动时会作为迁移源，之后不再写入） |

每个环境目录结构相同：

```
autoforge-{development|production}/
├── autoforge.db
├── script-inputs/
├── scripts/{scriptId}/
├── runtime/              # 全局 npm 依赖
└── runtime-python/       # 全局 Python 依赖
```

**迁移规则：**

1. 若存在旧版 `autoforge/` 且尚无 `-development` 数据 → 复制到 `autoforge-development/`
2. 若存在 `-development` 且尚无 `-production` 数据 → 复制到 `autoforge-production/`
3. 复制后重写数据库内 `workspace_path` 及配置 JSON 中的绝对路径

开发调试与正式使用的数据、脚本、执行历史**完全隔离**，互不影响。

## 运行确认

以下入口在触发执行前会弹出确认框「确定运行「脚本名」？」：

- 脚本详情面板 → **运行**
- 侧边栏脚本卡片 → 快捷运行

定时任务触发**不**经过此确认（与 v1.9.0 行为一致）。

---

## 开发与构建命令

```bash
npm run dev       # development mode → autoforge-development/
npm run build     # production mode 编译
npm run preview   # production mode 预览 → autoforge-production/
npm run prod      # build + preview 快捷组合
npm run dist:win  # 打包 NSIS 安装程序
```

环境变量解析优先级：`AUTOFORGE_APP_ENV` → `NODE_ENV` → `ELECTRON_RENDERER_URL`（dev server）→ `production`。
