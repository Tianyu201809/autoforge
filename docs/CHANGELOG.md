# 更新日志

本文件记录 Autoforge 桌面应用的版本变更。脚本包规范版本（`autoforge.json` 中的 `"autoforge": "1.0"`）与应用 semver 独立维护。

---

## [1.0.0] — 2026-06-28

首个正式版本。Autoforge 从 ScriptBox 原型演进为本机脚本锻造与管理平台，具备完整的脚本生命周期管理能力。

### 平台能力

- **脚本管理** — 导入脚本包或单文件，支持搜索、排序、收藏、归档与自定义分类
- **拖拽导入** — 将脚本目录或 `.js` / `.mjs` / `.cjs` 文件拖入主窗口即可导入
- **在线编辑** — 应用内编辑 `autoforge.json` 与入口代码
- **多环境 Profile** — 开发 / 测试 / 生产等环境变量集，脚本按环境独立配置
- **运行参数** — 文本参数与附件参数（`type: "attachment"`），支持多文件上传
- **运行与日志** — 一键运行、停止、重启；实时日志；运行结果展示
- **执行历史** — 按日汇总运行记录，支持搜索、状态筛选与 90 天自动清理
- **定时任务** — 六段 Cron 表达式（秒 分 时 日 月 周）自动触发
- **依赖管理** — 脚本级 npm 依赖 + 全局共享依赖
- **浏览器自动化** — 内置 Playwright bundle；脚本级无头模式开关
- **窗口模式** — 托盘常驻、桌面悬浮球、全局快捷键 `Ctrl+Shift+A`
- **主题** — 浅色 / 深色皮肤切换
- **示例与开发指南** — 内置示例脚本与完整脚本开发规范

### 脚本规范（autoforge 1.0）

- `env` / `params` 双维度输入模型
- 附件类型运行参数（JSON 数组，含 `name` / `path`）
- 脚本级 `browser.headless` 配置
- ScriptBox 旧格式自动迁移

### 技术栈

Electron 34 · Vue 3 · TypeScript · Tailwind CSS 4 · Playwright Core · node-cron

### 平台要求

- Windows x64（主要目标平台）
- Node.js 18+（开发构建）

### 已知限制

- 脚本在主进程 Node 环境运行，暂无子进程沙箱隔离
- 环境变量中的 secret 类型尚未加密存储
- 无脚本包版本历史与回滚
- 无单次运行超时与全局并发限制

详见 [v1.0.0 版本说明](./v1.0.0.md) 与 [架构说明](./architecture.md)。

## 历史开发阶段（1.0.0 之前）

以下为 MVP 至 RC 阶段的主要里程碑，未单独发版：

| 阶段 | 要点 |
|------|------|
| MVP | ScriptBox 原型、脚本导入与运行、环境 Profile |
| 参数维度 | 区分 `env` 与 `params` |
| 附件参数 | `type: "attachment"` 与 `script-inputs/` 缓存 |
| 无头模式 | 脚本级 `browser.headless` |
| 分类系统 | 内置 + 自定义分类管理 |
| 拖拽上传 | 主窗口拖放导入 |
| 多窗口模式 | 托盘、悬浮球、全局快捷键 |
| 执行历史 | `execution-history.json` 持久化与 UI 面板 |
