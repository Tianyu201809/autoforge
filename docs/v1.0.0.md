# Autoforge v1.0.0 版本说明

> 发布日期：2026-06-28 · 应用版本 `1.0.0` · 脚本规范 `autoforge 1.0`

Autoforge 是一个运行在 Windows 上的 Electron 桌面应用，作为**本机自动化脚本锻造与管理平台**。应用本身不包含业务逻辑；用户将脚本打包为标准目录导入后，即可在本机运行、配置环境、查看日志、设置定时任务并追溯执行历史。

---

## 设计原则

| 原则 | 说明 |
|------|------|
| 零内置脚本 | 平台与业务解耦，所有脚本由用户导入 |
| 脚本即包 | 每个脚本 = `autoforge.json` + 入口文件 + 可选 npm 依赖 |
| 配置与代码分离 | 脚本声明 schema，具体值由环境 Profile 与运行参数提供 |
| 生命周期可观测 | 每次运行有明确阶段，日志实时推送，结果与历史可追溯 |

---

## 功能清单

### 脚本管理

| 功能 | 说明 |
|------|------|
| 导入 | 脚本包目录、单文件（`.js` / `.mjs` / `.cjs`）、拖拽到主窗口 |
| 列表筛选 | 全部 / 运行中 / 收藏 / 最近 / 归档 |
| 分类 | 内置分类 + 自定义分类（侧边栏「管理分类」） |
| 搜索与排序 | 按名称搜索；按名称或最近运行排序 |
| 元数据编辑 | 名称、图标、分类、无头模式等（详情面板） |
| 在线编辑 | 应用内编辑入口代码与 `autoforge.json` |
| 归档 / 收藏 | 脚本卡片快捷操作 |

### 环境与参数

| 维度 | 用途 | 配置位置 | 脚本访问 |
|------|------|----------|----------|
| **环境变量 `env`** | 账号、Token、API 地址等固定配置 | 详情 → **配置** Tab | `ctx.env.KEY` |
| **运行参数 `params`** | 订单号、日期、任务 ID 等业务输入 | 详情 → **详情** Tab | `ctx.params.KEY` |
| **附件参数** | 每次运行需上传的本地文件 | 详情 → **详情** Tab | `ctx.params.KEY`（JSON 数组字符串） |

环境 Profile 在 **设置** 中管理；各脚本的 env 按环境独立保存，params 按脚本保存（不区分环境）。

### 运行与观测

```
queued → validating → installing-deps → starting → running → completed
                                                      ↘ failed
                                                      ↘ stopped（用户取消）
```

- **手动运行** — 详情 Tab 点击「运行」或「重启」
- **定时运行** — Cron 六段式：`秒 分 时 日 月 周`
- **实时日志** — 底部日志面板，`INFO` / `WARN` / `ERROR`
- **执行历史** — 侧边栏入口；按日分组，显示成功 / 失败 / 停止 / 运行中；支持脚本名搜索；默认展示近 30 天

触发来源标记为 `manual`（手动）或 `scheduled`（定时）。

### 浏览器自动化

- 内置 Playwright Chromium bundle（`resources/browsers/`，安装后复制到 userData）
- 脚本内调用 `ctx.sdk.browser.launch()`
- **无头模式** — 脚本详情 → 概览 →「浏览器无头模式」；也可在 `autoforge.json` 中声明 `"browser": { "headless": true }`
- 设置中可指定自定义浏览器可执行路径

> 若脚本需人工验证码或 2FA，请关闭无头模式，在弹出浏览器中手动完成。

### 依赖管理

| 类型 | 声明方式 | 安装位置 |
|------|----------|----------|
| 脚本级 | `autoforge.json` → `dependencies` | `userData/scripts/{id}/node_modules/` |
| 全局共享 | 设置 → 全局 npm 依赖 | `userData/runtime/node_modules/` |

首次运行前自动 `npm install`；全局依赖对所有脚本可见。

### 窗口与快捷操作

| 模式 | 开启方式 | 行为 |
|------|----------|------|
| 托盘模式 | 设置 → 窗口行为 | 关闭窗口时隐藏到系统托盘，后台继续运行 |
| 悬浮球 | 设置或标题栏按钮 | 桌面圆形悬浮球，拖动定位，点击打开主界面；右键菜单 |
| 全局快捷键 | 设置 → 窗口行为 | `Ctrl+Shift+A` 快速显示主界面 |
| 主题 | 标题栏或设置 | 浅色 / 深色切换 |

### 开发指南

应用内 **开发指南** 面板提供：

- 一键导入内置示例（`hello-world` 等）
- 阅读完整脚本开发规范（打包时随安装包分发 `docs/script-spec.md`）

## 本机数据目录

默认位于 Electron `userData` 路径（Windows 通常为 `%APPDATA%/autoforge/`）：

```
userData/
├── autoforge-data.json       # 脚本注册表、环境 Profile、分类、应用配置
├── execution-history.json    # 执行历史（最多 5000 条，保留 90 天）
├── script-inputs/            # 运行参数附件缓存
├── scripts/
│   └── {scriptId}/
│       ├── autoforge.json
│       ├── index.mjs
│       ├── package.json      # 依赖安装时自动生成
│       └── node_modules/
└── runtime/
    ├── package.json          # 全局共享依赖
    └── node_modules/
```

应用重启后，执行历史中状态为 `running` 的记录会自动标记为 `stopped`。

## 脚本包最小示例

```
my-script/
├── autoforge.json
└── index.mjs
```

**autoforge.json**

```json
{
  "autoforge": "1.0",
  "name": "我的脚本",
  "description": "脚本说明",
  "version": "1.0.0",
  "entry": "index.mjs",
  "browser": { "headless": false },
  "env": [
    { "key": "API_URL", "label": "API 地址", "required": true }
  ],
  "params": [
    { "key": "ORDER_ID", "label": "订单号", "required": true }
  ]
}
```

**index.mjs**

```javascript
export async function run(ctx) {
  ctx.log('INFO', `处理订单 ${ctx.params.ORDER_ID} @ ${ctx.env.API_URL}`)
  return { ok: true }
}
```

完整字段说明见 [脚本开发规范](./Autoforge脚本开发规范文档说明.md)。

## 安装与构建

### 环境要求

- Node.js 18+
- Windows x64

### 从源码运行

```bash
git clone <repo-url>
cd Autoforge
npm install          # postinstall 自动下载 Playwright 浏览器
npm run dev
```

### 打包安装程序

```bash
npm run dist:win     # 输出至 release/
```

## 迁移说明

从旧版 **ScriptBox** 升级：

- 首次保存时 `script-box-data.json` 自动迁移至 `autoforge-data.json`
- 旧版 `scriptbox.json` 清单仍可导入，保存后写入 `autoforge.json`
- 示例 `hello-scriptbox` 演示兼容导入流程

## 后续规划

v1.0.0 尚未包含、计划在后续版本考虑的能力：

| 能力 | 优先级 |
|------|--------|
| 脚本沙箱（子进程隔离） | 高 |
| 密钥保险库（secret 加密存储） | 高 |
| 单次运行超时 | 中 |
| 并发限制 | 中 |
| 脚本包版本历史与回滚 | 中 |
| 导入校验 UI | 中 |
| 脚本模板脚手架 | 中 |
| 脚本市场 | 低 |

详见 [架构说明](./architecture.md)。
