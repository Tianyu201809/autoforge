# Autoforge v1.9.0 版本说明

> 发布日期：2026-07-03 · 应用版本 `1.9.0` · 脚本规范 `autoforge 1.0`

Autoforge 是运行在 Windows 上的 Electron 桌面应用，作为**本机自动化脚本锻造与管理平台**。应用本身不包含业务逻辑；用户将 JavaScript 或 Python 脚本打包为标准目录导入后，即可在本机运行、配置环境、查看日志、设置定时任务并追溯执行历史。

---

## 相对 v1.0.0 的主要变化

| 领域 | 新增 / 变更 |
|------|-------------|
| **脚本语言** | 支持 Python 3.9+（子进程运行）；JS 与 Python 共用 `run(ctx)` 契约 |
| **持久化** | JSON 文件存储迁移至 SQLite（`autoforge.db`） |
| **进度回显** | 脚本可通过 `ctx.stage` / `ctx.progress` 推送阶段与进度，UI 实时展示 |
| **Schema 类型** | 环境变量与运行参数支持多类型字段（文本、数字、布尔、选项等） |
| **运行控制** | 可配置单次运行超时（秒）；依赖安装结果缓存 |
| **界面** | 6 套深浅皮肤、脚本列表分页、小记面板、执行历史详情弹窗 |
| **入口函数** | JS 脚本支持 `run` / `main` / `default`；Python 支持 `run` / `main` |
| **开发体验** | 内置 `autoforge-script-create` Skill；脚本规范 Markdown 可下载 |

---

## 设计原则

| 原则 | 说明 |
|------|------|
| 零内置脚本 | 平台与业务解耦，所有脚本由用户导入 |
| 脚本即包 | 每个脚本 = `autoforge.json` + 入口文件 + 可选依赖 |
| 配置与代码分离 | 脚本声明 schema，具体值由环境 Profile 与运行参数提供 |
| 生命周期可观测 | 每次运行有明确阶段，日志与进度实时推送，结果与历史可追溯 |

## 功能清单

### 脚本管理

| 功能 | 说明 |
|------|------|
| 导入 | 脚本包目录、单文件（`.js` / `.mjs` / `.cjs` / `.py`）、拖拽到主窗口 |
| 语言 | JavaScript（主进程动态加载）与 Python（独立子进程） |
| 列表筛选 | 全部 / 运行中 / 收藏 / 最近 / 归档 |
| 分页 | 脚本列表分页展示（每页 12 条） |
| 排序 | 按名称、最近运行或上传时间 |
| 分类 | 内置分类 + 自定义分类 |
| 在线编辑 | 应用内编辑入口代码与 `autoforge.json` |
| 归档 / 收藏 | 脚本卡片快捷操作 |

### 环境与参数

| 维度 | 用途 | 配置位置 | 脚本访问 |
|------|------|----------|----------|
| **环境变量 `env`** | 账号、Token、API 地址等固定配置 | 详情 → **配置** Tab | `ctx.env.KEY` |
| **运行参数 `params`** | 订单号、日期、任务 ID 等业务输入 | 详情 → **详情** Tab | `ctx.params.KEY` |
| **附件参数** | 每次运行需上传的本地文件 | 详情 → **详情** Tab | `ctx.params.KEY`（JSON 数组字符串） |

环境 Profile 在 **设置** 中管理；各脚本的 env 与 params 按环境独立保存。

### 运行与观测

```
queued → validating → installing-deps → starting → running → completed
                                                      ↘ failed
                                                      ↘ stopped（用户取消 / 超时）
```

- **手动运行** — 详情 Tab 点击「运行」或「重启」
- **定时运行** — Cron 六段式：`秒 分 时 日 月 周`；卡片显示执行周期摘要
- **运行超时** — 设置 → Python → 运行超时（秒），`0` 为不限制
- **实时日志** — 终端面板，`INFO` / `WARN` / `ERROR`
- **进度回显** — 脚本调用 `ctx.stage()` / `ctx.progress()`，UI 展示阶段与进度条
- **执行历史** — 侧边栏入口；按日分组；点击条目查看详情弹窗；默认展示近 30 天

### 浏览器自动化

- 内置 Playwright Chromium bundle（`resources/browsers/`）
- JS：`ctx.sdk.browser.launch()`；Python：同名 SDK，需 `dependencies` 含 `playwright`
- **无头模式** — 脚本详情或 `autoforge.json` 中 `"browser": { "headless": true }`

### 依赖管理

| 类型 | 语言 | 声明方式 | 安装位置 |
|------|------|----------|----------|
| 脚本级 | JS | `dependencies`（npm） | `userData/scripts/{id}/node_modules/` |
| 脚本级 | Python | `dependencies`（pip） | `userData/scripts/{id}/.venv/` |
| 全局共享 | JS | 设置 → 全局 npm 依赖 | `userData/runtime/node_modules/` |
| 全局共享 | Python | 设置 → Python → 全局 Python 依赖 | `userData/runtime-python/.venv/` |

依赖安装结果会缓存，未变更时跳过重复安装。Python 可在设置中配置 pip 镜像源。

### Python 运行时

- 使用本机 Python 3.9+（**设置 → Python** 可指定 `python.exe` 路径）
- 打包时随应用分发 `autoforge_runtime` 包（`resources/python/`）
- Python 脚本在独立子进程中运行；停止运行会终止子进程
- 示例：`examples/hello-world-py/`、`examples/playwright-py/`

### 窗口与界面

| 能力 | 说明 |
|------|------|
| 托盘模式 | 关闭窗口时隐藏到系统托盘 |
| 悬浮球 | 桌面圆形悬浮球，拖动定位，点击打开主界面 |
| 全局快捷键 | 默认 `Ctrl+Shift+A` 显示主界面；支持动态注册 |
| 皮肤 | 6 套预设（锻炉 / 曜石 / 松野 / 暖纸 / 素纸 / 浅樱），深浅成对切换 |
| 小记 | 标题栏「小记」面板，保存常用片段，一键插入到当前输入框 |
| 版本号 | 设置页显示当前应用版本 |

### 开发指南

- 应用内 **开发指南** 面板：一键导入示例、阅读脚本规范
- 可下载 `script-spec.md`（Markdown 格式脚本开发规范）
- 内置 Skill：`skills/autoforge-script-create/`（随安装包分发）

## 本机数据目录

> **v1.10.0 起** userData 按启动方式隔离，见 [v1.10.0 版本说明](./v1.10.0.md)。以下为 v1.9.0 及更早的扁平布局说明。

默认位于 Electron `userData` 路径（Windows 通常为 `%APPDATA%/autoforge/`）：

```
userData/
├── autoforge.db              # SQLite：脚本、环境、分类、配置、执行历史
├── *.migrated.bak            # 从旧版 JSON 迁移时的备份（如有）
├── script-inputs/            # 运行参数附件缓存
├── scripts/
│   └── {scriptId}/
│       ├── autoforge.json
│       ├── index.mjs | index.py
│       ├── package.json      # JS 依赖（自动生成）
│       ├── node_modules/     # JS 脚本依赖
│       └── .venv/            # Python 脚本依赖
├── runtime/
│   ├── package.json
│   └── node_modules/         # 全局 npm 依赖
└── runtime-python/
    └── .venv/                # 全局 Python 依赖
```

应用重启后，执行历史中状态为 `running` 的记录会自动标记为 `stopped`。

## 脚本包最小示例

### JavaScript

```
my-script/
├── autoforge.json
└── index.mjs
```

```json
{
  "autoforge": "1.0",
  "name": "我的脚本",
  "entry": "index.mjs",
  "env": [{ "key": "API_URL", "label": "API 地址", "required": true }],
  "params": [{ "key": "ORDER_ID", "label": "订单号", "required": true }]
}
```

```javascript
export async function run(ctx) {
  ctx.log('INFO', `处理订单 ${ctx.params.ORDER_ID}`)
  return { ok: true }
}
```

### Python

```
my-py-script/
├── autoforge.json
└── index.py
```

```json
{
  "autoforge": "1.0",
  "name": "我的 Python 脚本",
  "language": "python",
  "entry": "index.py"
}
```

```python
async def run(ctx):
    ctx.log("INFO", "开始执行")
    return {"ok": True}
```

完整字段说明见 [脚本包规范](./script-spec.md)。

## 安装与构建

### 环境要求

- Node.js 18+
- Windows x64（主要目标平台）
- Python 3.9+（仅运行 Python 脚本时需要）

### 从源码运行

```bash
git clone <repo-url>
cd Autoforge
npm install          # postinstall 自动下载 Playwright 浏览器
npm run dev
```

### 打包安装程序

```bash
npm run dist:win     # 输出至 release/
```

## 迁移说明

- **ScriptBox → Autoforge**：旧版 `scriptbox.json` 仍可导入，保存后写入 `autoforge.json`
- **JSON → SQLite**：v1.2+ 首次启动时，若存在 `autoforge-data.json` / `execution-history.json` 等旧文件，会自动导入 `autoforge.db` 并重命名为 `*.migrated.bak`

---

## 已知限制

- JavaScript 脚本仍在主进程 Node 环境运行，暂无子进程沙箱隔离
- Python 脚本依赖用户本机解释器，应用不捆绑 Python 运行时
- 环境变量中的 secret 类型尚未加密存储
- 无脚本包版本历史与回滚
- 无全局并发运行数量限制
