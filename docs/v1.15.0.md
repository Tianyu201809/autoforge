# Autoforge v1.15.0 版本说明

> 发布日期：2026-07-11 · 应用版本 `1.15.0` · 脚本规范 `autoforge 1.0`

Autoforge 是本机自动化脚本锻造与管理桌面应用。v1.15.0 新增与 **Autoforge Hub** 的本机桥接：在 Hub 网站点击「添加到本地」后，可将脚本 zip 一键导入本机 Autoforge。

完整功能基线见 [v1.9.0](./v1.9.0.md)；近期 UI / 数据隔离见 [v1.11.0](./v1.11.0.md)、[v1.10.0](./v1.10.0.md)。本文档仅描述 v1.15.0 新增与变更。

设计规格：

- 桌面端：[2026-07-11-hub-local-install-design.md](./superpowers/specs/2026-07-11-hub-local-install-design.md)
- Hub 端（独立实现）：[2026-07-11-hub-local-install-hub-side-design.md](./superpowers/specs/2026-07-11-hub-local-install-hub-side-design.md)

## 相对 v1.14.0 的变化

| 领域 | 变更 |
|------|------|
| **Hub 一键安装** | 应用运行时在 `127.0.0.1:19276` 提供本机 HTTP 桥；Hub 可探测并请求安装公开 zip |
| **导入流水线** | 下载 zip → 解压 → 校验 `autoforge.json` → 复用现有 `script-registry` 导入 |
| **安装成功体验** | 主窗口前置，选中并打开新脚本，toast 提示 |

## Hub 一键安装到本地

### 前提

1. 本机 **已启动** Autoforge（未启动时 Hub 应提示先打开桌面端；本期不自动拉起）
2. Hub 提供可公开下载的脚本 **zip**（内含合法 `autoforge.json` 与入口文件）
3. v1 **不做** zip 鉴权；重复安装会得到新的脚本副本

### 本机桥 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `http://127.0.0.1:19276/health` | 探测是否在线 |
| `POST` | `http://127.0.0.1:19276/install` | body：`{ zipUrl, scriptName?, hubScriptId? }` |

仅监听 `127.0.0.1`。`zipUrl` 须为绝对 `http:` / `https:` URL。

### 用户路径

```
Hub「添加到本地」
  → GET /health（失败则提示启动 Autoforge）
  → POST /install { zipUrl }
  → 桌面端下载、导入、聚焦窗口并打开脚本
```

侧边栏「脚本市场」仍为占位；本期入口在 **Hub 网站**，不在应用内目录。

### 模块

| 模块 | 职责 |
|------|------|
| `hub-bridge-server` | localhost HTTP、CORS、安装锁、`/health` `/install` |
| `hub-script-installer` | 下载、解压、定位包根、调用 `scriptRegistry.importFromPath` |
